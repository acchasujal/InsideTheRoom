# Screen Specifications

## 1. Home / Hook Screen
- **Purpose:** Set the thesis and reframe expectations.
- **User Actions:** Click "Enter the VAR Room".
- **Inputs:** None.
- **Outputs:** The thesis statement: "Football's biggest controversies aren't about what happened. They're about what a word means."
- **Success Condition:** User proceeds to the first incident with an understanding that this is about language, not just data.

## 2. Incident Setup Screen (Reused per incident)
- **Purpose:** Present the incident neutrally.
- **User Actions:** Watch video/read text, then click to make Decision 1.
- **Inputs:** None.
- **Outputs:** Neutral summary of the incident (e.g., Perišić handball).
- **Success Condition:** User understands what happened physically on the pitch.

## 3. Decision 1 Screen (Reused per incident)
- **Purpose:** Lock in the user's initial, biased judgment (Experience before Explanation).
- **User Actions:** Select an outcome (e.g., "Penalty" or "No Penalty").
- **Inputs:** Button click for the decision.
- **Outputs:** Display of the user's choice.
- **Success Condition:** User commits to a stance based on their current understanding.

## 4. Perspectives Screen (Reused per incident)
- **Purpose:** Demonstrate Granite's multi-perspective reasoning capability.
- **User Actions:** Scroll/swipe through 2-3 starkly diverging perspectives.
- **Inputs:** None.
- **Outputs:** Granite-generated arguments (Fan, Referee, VAR, etc.) grounded in evidence.
- **Success Condition:** User sees that the exact same evidence can logically support opposite conclusions.

## 5. Reveal Screen (Reused per incident)
- **Purpose:** Expose the structural ambiguity in FIFA Law.
- **User Actions:** Read the specific law text and the identified undefined term.
- **Inputs:** None.
- **Outputs:** Exact FIFA Law citation with highlighted undefined terms (e.g., "deliberately").
- **Success Condition:** User realizes the law lacks an objective definition.

## 6. Decision 2 Screen (Reused per incident)
- **Purpose:** Force reflection on the user's initial certainty.
- **User Actions:** Select an outcome again or answer a reflection prompt.
- **Inputs:** Button click.
- **Outputs:** Comparison of Decision 1 certainty vs. Decision 2 reality.
- **Success Condition:** User acknowledges that their first decision was built on undefined terms.

## 7. Climax / Suárez Screen
- **Purpose:** Deliver the emotional climax regarding the law's deliberate silence.
- **User Actions:** Answer "Was justice served?"
- **Inputs:** Button click.
- **Outputs:** The reveal that the law defines procedure, not justice.
- **Success Condition:** User feels the emotional weight of a correct procedural outcome failing moral context.

## 8. Live Generation Screen (Layer 2)
- **Purpose:** Prove Granite is structurally necessary by processing novel inputs.
- **User Actions:** Type a custom incident description.
- **Inputs:** Free-text incident description.
- **Outputs:** Live LangFlow pipeline execution resulting in 5 divergent perspectives.
- **Success Condition:** Judge witnesses real-time, non-trivial AI synthesis.
